## [2.5.0] - 2026-02-02

### Added

#### CLI - Project Core Commands (`wiz project`)
- **`build`**: Build project with Angular compilation
  - `--project`: Target project name (default: main)
  - `--clean`: Clean cache before build
- **`create`**: Create new project
  - `--project`: Project name
  - `--uri`: Clone from Git repository
  - `--path`: Import from .wizproject file
- **`delete`**: Delete project with confirmation
- **`list`**: List all projects with status
- **`clean`**: Clean project cache
- **`export`**: Export project to .wizproject file
  - `--output`: Output file path

#### CLI - App Management (`wiz project app`)
- **`list`**: List all apps in project
- **`create`**: Create new app component
  - `--app`: App ID (e.g., page.home)
  - `--engine`: Template engine (html/pug)
- **`delete`**: Delete app component

#### CLI - Controller Management (`wiz project controller`)
- **`list`**: List all controllers
- **`create`**: Create new controller
  - `--controller`: Controller name
  - `--code`: Initial code content
- **`delete`**: Delete controller

#### CLI - Route Management (`wiz project route`)
- **`list`**: List all custom routes
- **`create`**: Create new route
  - `--route`: Route name
  - `--path`: URL path pattern
- **`delete`**: Delete route

#### CLI - Package Management (`wiz project package`)
- **`list`**: List all portal packages
- **`export`**: Export package to .portal file
- **`import`**: Import package from .portal file

#### CLI - NPM Management (`wiz project npm`)
- **`install`**: Install npm packages in build folder
  - `--package`: Package name(s)
- **`uninstall`**: Uninstall npm packages
- **`list`**: List installed packages

#### Documentation
- **[docs/kr/command.md](docs/kr/command.md)**: Complete Korean command reference
- **[docs/en/command.md](docs/en/command.md)**: Complete English command reference
- **WebSocket Guide**: Comprehensive socket.py documentation with Controller class structure, SocketHandler API, and practical examples

### Changed
- **[core]** `wiz.ide.plugin.model` and `command` methods updated to support `mode='context'` for CLI context injection without performance overhead in web mode
- **[cli]** Refactored CLI architecture with modular subcommand structure
- **[cli]** All command help messages and comments converted to English for international compatibility
- **[builder]** Improved build process stability with better error handling
- **[npm]** NPM operations now use build folder with automatic sync

### Fixed
- **[cli]** Fixed path resolution for recursive project root finding and symlink handling
- **[cli]** Fixed dependency injection issues in plugin execution within CLI environment
- **[builder]** Fixed missing directory handling during build process

## MCP & Agent Integration

The v2.5 CLI commands are designed to work seamlessly with AI agents and MCP servers:

```bash
# AI agents can manage projects programmatically
wiz project list                              # Discover projects
wiz project app list --project=main           # List components
wiz project app create --project=main --app=page.new --engine=pug
wiz project build --project=main --clean      # Build project

# Export/Import for project sharing
wiz project export --project=main --output=backup.wizproject
wiz project create --project=restored --path=backup.wizproject
```

All commands provide structured output suitable for machine parsing and automation.
